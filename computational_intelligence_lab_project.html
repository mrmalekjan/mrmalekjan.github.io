Computational Intelligence Lab Project: Persian Voice Command Classification for Smart Home Devices

Course: Computational Intelligence Laboratory

Technology Stack: Python, TensorFlow, Raspberry Pi, Audio Processing

Key Features: Persian voice recognition, Smart home control, Deep learning classification

Project Overview

This Computational Intelligence Lab project focused on developing a deep learning model for Persian voice command classification to control smart home devices. The system processes voice commands in Persian language and uses them to control electrical switches in a smart home environment.

The project addresses the growing importance of voice processing in machine learning and its applications in IoT systems, similar to popular voice assistants like Siri and Google Voice Command.

System Architecture

Inputs & Outputs

  • Inputs: Raspberry Pi microphone, temperature sensor, light sensor
  • Outputs: Control of two simple electrical switches (ON/OFF)

Voice Command Recognition

The neural network was designed to recognize the following Persian voice commands:

  • Automatic/Manual Mode: "خودکار" (automatic), "دستی" (manual)
  • Temperature Control: "گرم" (warm), "سرد" (cold)
  • Lighting Control: "تاریک" (dark), "روشن" (bright)

Implementation Methodology

1. Data Collection and Preparation

  • Created a custom Persian voice dataset due to limited available resources
  • Used Python libraries for audio recording and processing
  • Audio files saved in WAV format with 2-3 second duration
  • Implemented data augmentation with 20% noise addition for better generalization

2. Audio to Matrix/Image Conversion

  • Converted audio files to suitable formats for classification using scipy.io.wavfile or soundfile libraries
  • Transformed audio data into MFCC (Mel-Frequency Cepstral Coefficients) or spectrogram representations
  • Prepared feature matrices for deep learning model input

3. Deep Learning Model Design

  • Designed a multi-layer neural network for voice command classification
  • Implemented Convolutional Neural Network (CNN) architecture suitable for audio pattern recognition
  • Used TensorFlow/Keras framework for model development and training
  • Dataset split into training, validation, and testing subsets
  • Trained final model saved for deployment

4. Raspberry Pi Deployment

  • Deployed the trained model on Raspberry Pi for real-time operation
  • Integrated voice command processing with electrical switch control
  • Implemented LED control based on voice command classification

Operational Modes

Automatic Mode

  • System reads sensor data (temperature and light) automatically
  • Maintains temperature and light at predefined optimal levels
  • No voice commands required for basic operation

Manual Mode

  • Responds to Persian voice commands for temperature and lighting control
  • Commands "warm/cold" and "dark/bright" adjust environmental conditions based on user preference
  • Provides user-driven control of smart home environment

Technical Challenges & Solutions

  • Challenge: Limited Persian voice dataset availability
  • Solution: Created custom dataset with proper data augmentation techniques
  • Challenge: Audio feature extraction and preprocessing
  • Solution: Implemented MFCC and spectrogram conversion for effective pattern recognition
  • Challenge: Real-time processing on Raspberry Pi
  • Solution: Optimized model architecture and used efficient TensorFlow implementation
